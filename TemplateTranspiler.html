TemplateTranspiler module¶

class TemplateTranspiler.TemplateTranspiler(data_type='float', record_filename='data.codegen.discrete_records.record_1', loop_type='config', code_output_directory=None, output_directory='../data/exec/output')[source]¶

Bases: src.codegen.Transpiler.Transpiler

Abstract base class for for transpilers generating code for a specific language by modifying a template.

Subclasses are required to implement all abstract functions and properties (see comments below).

__init__(data_type='float', record_filename='data.codegen.discrete_records.record_1', loop_type='config', code_output_directory=None, output_directory='../data/exec/output')[source]¶

Base constructor for TemplateTranspiler objects. Extends the constructor of base class Transpiler.

Parameters

  • data_type (str) – Datatype that should be used in the code. Default is ‘float’.

  • record_filename (types.ModuleType) – Module containing the discretized ODE system.

  • loop_type (str) – Type of the loop used for iteration in implicit integration methods (ignored otherwise). Default is ‘config’. Other possibilities: ‘for’, ‘both’, ‘while’, ‘counter’.

  • code_output_directory (str) – Path to the directory where the generated code files should be saved. Default is None.

  • output_directory (str) – Path of the directory of the default file for the integrated solution. Default is ‘../data/exec/output’.

abstract property marker¶

Marker that indicates a part of the template that needs to be replaced by generated code.

These parts begin and end with marker.

Example

marker = '##' causes ## example ## to be replaced by the value of config['example'].

generate()[source]¶

Generate code implementing the system specified in self.record.

This is done loading code from the template files and replacing the sections marked with self.marker with data from self.config (discretized system) and self.module_file

generate_loop(curr_eq, loop_type, loop_iterations, condition)[source]¶

Generate code to implement the loop specified in the parameters.

The generated loop is a loop of type loop_type with a loop condition depending on loop_iterations and condition and loop_content as the body. This is an abstract method, which has to be implemented by every subclass of Transpiler. This method should implement the code generation for iterative equations in the corresponding language.

Parameters

  • loop_content (sympy.codegen.ast.While) – Sympy while loop with the loop body being a block containing sympy expressions.

  • loop_type (str) – Type of the generated loop.

  • loop_iterations (int) – Number of times the loop will be executed (if it is iterative).

  • condition (sympy.logic.boolalg.Boolean) – Sympy expression for the iteration condition of the loop (if applicable), e.g. i<5.

Returns

A string with the code.
