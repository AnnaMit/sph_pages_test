This abstract class is a template for time integration Methods. This class itself cannot be instantiated. It ensures, that every Method contains a term, the timestep length and for convinience a list of all variables.

class TimeIntegratorUnit.MultiDependencyTimeIntegratorUnit(terms, variables)[source]¶

Bases: TimeIntegratorUnit.TimeIntegratorUnit

This is the base template class for integration methods that heavily depend on multiple equations of the ODE. Instances of these class have a variable called terms, which stores all equations the integration needs, due to dependency. The variable lhs_list stores the variables of the left hand sides of the original ODE (i.e. u(t)/dt) corresponding to the order of the equations in self.terms. The time_tendencies function has to be rewritten, to be able to index the right term.

class TimeIntegratorUnit.Heun(terms, variables)[source]¶

Bases: TimeIntegratorUnit.MultiDependencyTimeIntegratorUnit

Heun’s Method is a two-stage second order Runge Kutta method, which is often referred to as the improved Euler’s method. It should always hold, that Midpoint.adapt_term == RungeKutta.adapt_term, when RungeKutta is called with the following Butcher Tableau:

0
1  1
-  1/2  1/2

Its general solution is as follows: ỹ_nplus1 = y_n + deltaT * f(t_n, y_n) y_nplus1 = y_n + deltaT/2 *(f(t_n,y_n) + f(t_nplus1, ỹ_nplus1)

class TimeIntegratorUnit.RungeKutta(terms, variables)[source]¶

Bases: TimeIntegratorUnit.MultiDependencyTimeIntegratorUnit

Runge Kutta is a class of integration methods, whose members are all of form: \(y_{n+1} = y_n + dT * Σ b_i*k_i\) with \(k_i = f(t_n + c_i * dT, y_n + a_ii * k_i-1)\). A Butcher Table, can be created for each specific Runge Kutta, which has the form:

0
c_2    a_21
c_3    a_31    a_32
...    ...     ...     ...
c_s    a_s1    a_s2    ...     a_s,s-1
 -     b_1     b_2     ...     b_s-1   b_s

This class only represents explicit Runge Kutta methods, which implies, that the Butcher Tableau is a strict lower tridiagonal matrix. Note that this class is not intended to be instantiated. Instead create object off subclasses and use the compute_stages method to get a discrete form depending on a butcher tableau.

compute_stages(ties, butcher_tab, dist_code)[source]¶
Parameters

  • butcher_tab – The Butcher-Tableau defining the member of the Runge-Kutta class. This should be a tuple containing three lists, that define vector c, matrix A and vector b respectively.

  • dist_code – A distinct String, which defines the member of the Runge-Kutta class, that is used. This is important, to not confuse the Runge-Kutta Stages of different butcher-Tableaus.

This method will append every RungeKutta Stage of every equation in self.terms to the TIES object. It will also append the equation describing the integrated form of y_nplus1, where y_nplus1 is correspondig to the variable at self.lhs_list[0]. On the opposite to earlier versions of this function, the RungeKutta Stages will not be inserted into each higher Stage and thus save memory space as each Stage will now appear only once as a symbolic term.

class TimeIntegratorUnit.Midpoint(terms, variables)[source]¶

Bases: TimeIntegratorUnit.MultiDependencyTimeIntegratorUnit

The midpoint method is a second-order one-step method from the class of RungeKutta methods. The difference to the explicit Euler is in the slope, which is the tangent to the original function at the point in the middle between t and t+1. It should always hold, that Midpoint.adapt_term == RungeKutta.adapt_term, when RungeKutta is called with the following Butcher Tableau:

0
1/2  1/2
-     0    1

The general midpoint method to a problem of the form \(y'(t) = f(t,y(t))\) is the following: \(y_{n+1} = y_n + dt*f(t+ dt/2 , y_n + (dt/2) * f(t, y_n))\)

Source:

John Burkhardt, Florida State University : Introduction to Scientific Computing with C++, 2011 SS, Lecture 19, Slide 10

class TimeIntegratorUnit.StrangSplitting(annot_list, variables)[source]¶

Bases: TimeIntegratorUnit.OperatorSplitting

Strang splitting is used to divide an equation consisting of an addition of two terms, which are mostly not stable to the same integration method. Thus both terms are treated seperately. If the first term gets integrated for deltaT/2, then used as initialvalue for integrating the second term for deltaT, which then again is used to integrate the first term for deltaT/2, the error this method induces is only of second order.

Example

Equation:

\(u_t = λ_1 u + λ_2 u\)

Split into TimeIntegratorUnits:
→ tiu1 = ExplicitEuler(\(λ_1 u\))
→ tiu2 = ExplicitEuler(\(λ_2 u)\)
Discretized Equations (h⁼dT):
\(v = u_n + h / 2 * (λ_1 u_n)\)
\(w = v + h * (λ_2 v)\)
\(u_{n+1} = w + h/2 * (λ_1 w)\)

class TimeIntegratorUnit.BackwardEuler(terms, variables)[source]¶

Bases: TimeIntegratorUnit.DependentSystemTimeIntegratorUnit

Basic implementation of the Backwards/Implicit Euler method:

\(t_{n+1} = t_n + dT\) and \(x_{n+1}= x_n + dT * f(t_{n+1}, x_{n+1})\)

To approximate a solution \(x_{n+1}\), Newton’s method is used:

\(y_{i+1} = y_i - g(y_i)/g'(y_i)\)

Combining those, the algorithm generates the value at the next time step through iteration of the following formula until convergence:

\(y_{i+1} = y_i - (y_i - x_n - dT * f(t_{n+1}, y_n)) / (1 - dT * d/dt f(t_{n+1}, y_n))\)

The Backwards Euler Method can be seen as a first order single stage order Runge Kutta method with the following Butcher Tableau:

1 1
1
